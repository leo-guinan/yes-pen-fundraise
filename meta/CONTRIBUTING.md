# Contributing Guidelines

## How to Contribute

### For Scouts

1. **Fork the repository**
2. **Read the [Scouting Manifesto](../00-charter/scouting-manifesto.md)**
3. **Use the templates** — don't create from scratch
4. **Submit a PR** — with your evaluation or artifact
5. **Wait for review** — we'll review and merge

### For Contributors

1. **Open an issue** — for questions or suggestions
2. **Submit a PR** — for improvements to documentation
3. **Follow the structure** — don't break the organization
4. **Be respectful** — see Code of Conduct

## What "Good" Looks Like

### Good Evaluations

- Cover all 5 axes thoroughly
- Include specific examples and evidence
- Ask thoughtful questions
- Provide actionable suggestions
- Are well-written and clear

### Good Artifacts

- Are original and useful
- Are well-designed and clear
- Add real value to the Yes Room
- Can be referenced by others
- Stand the test of time

### Good Documentation Improvements

- Fix typos and errors
- Improve clarity
- Add missing information
- Organize better
- Link related content

## Etiquette

- **Be respectful** — no harassment, no trolling
- **Be constructive** — criticism is fine, but be helpful
- **Be honest** — don't shill, don't hide truth
- **Be patient** — reviews take time
- **Be collaborative** — help others improve

## What Not to Do

- Don't spam or troll
- Don't violate the Code of Conduct
- Don't submit low-effort content
- Don't break the repository structure
- Don't hide behind anonymity to be mean

## Review Process

1. **PR submitted** — scout or contributor submits PR
2. **Review** — founder or core team reviews
3. **Feedback** — if needed, feedback is provided
4. **Merge** — once approved, PR is merged
5. **Points awarded** — if applicable, points are added to leaderboard

---

**Thank you for contributing. Your work makes the Yes Room better.**
